# 数据获取

#### 1. 【爬虫内核】**Playwright > Selenium > Requests**

✅ **首选 Playwright**（核心推荐）

- 原因：招聘站多为**动态渲染（Vue/React）**，Requests 爬不到异步加载的职位信息；Playwright 比 Selenium 轻量、反爬友好，支持自动等待、模拟真实浏览器（Chrome/Firefox），还能一键截图 / 获取完整 DOM，完美适配各大厂商招聘站（阿里 / 腾讯 / 字节 / 华为 / 小米等）；
- 关键优势：自带**隐式等待、页面加载控制、反爬规避**（模拟鼠标 / 键盘、随机 UA），开发量比 Selenium 少 50%。

#### 2. 【自动翻页】**「DOM 关键词过滤 + 本地轻量大模型（Ollama）」> Web Research MCP**

✅ 首选**自研轻量方案**（适配招聘站场景，免费、省 Token、可控）

- 原因：招聘站的翻页结构**高度标准化**（底部分页栏、下一页按钮、加载更多），无需复杂的 Web Research MCP，用「BeautifulSoup 做 DOM 过滤（只保留分页节点）+ Ollama 本地模型（qwen2/llama3）识别翻页选择器」，**100% 本地运行，无 API 费用，Token 消耗几乎为 0**；
- 备用：如果部分厂商招聘站翻页结构特殊，再叠加 Web Research MCP 做精准解析（兜底）。

#### 3. 【AI 职位信息提取】**Ollama 本地大模型（首选） + 轻量 Prompt**

✅ 核心：**只喂「职位列表的迷你 HTML 片段」给本地 AI，不喂全量页面**

- 模型选择：Ollama 部署`qwen2:1.8b/7b`或`llama3:8b`（显存占用低，你的机器完全能跑，推理快，专门做结构化提取）；
- 替代：如果追求极致精准，可用 GPT-4o-mini / 文心一言轻量版（按 Token 收费，价格极低，单条职位提取仅需几十 Token）；
- 关键：招聘站的职位信息（职位名、岗位职责、任职要求、薪资、地点、发布时间）是**结构化 / 半结构化**的，AI 提取准确率能到 98%+，比纯正则 / XPATH 更通用（适配不同厂商的职位排版）。

#### 4. 【HTML 解析 / 轻量化】**BeautifulSoup4 + lxml**

- 必配组合：lxml 作为解析器（速度快），BeautifulSoup 做节点遍历 / 过滤，专门用来**提取「职位列表模块」和「分页模块」的迷你 HTML**，剔除广告 / 导航 / 无关内容，让 AI 只分析核心片段（省 Token、提准确率）。

#### 5. 【存储层】**MySQL + CSV（兜底）**

- 主存储：MySQL（结构化存储职位信息，方便后续查询 / 筛选 / 可视化，适配多厂商职位数据的统一 schema）；
- 兜底：CSV（爬取过程中实时写入，防止程序崩溃丢失数据，开发量为 0）；
- 可选：MongoDB（如果需要存储非结构化的岗位职责 / 任职要求，无需提前建表）。

#### 6. 【反爬 / 稳爬层】**Playwright 自带能力 + 轻量反爬策略**

招聘站的反爬**远低于电商 / 知乎**，无需复杂代理，简单策略就能稳爬：

- 随机 UA / 浏览器指纹；
- 随机翻页间隔（1-3 秒）；
- 模拟鼠标滚动（加载异步职位列表）；
- 单 IP 单站点低频率爬取（招聘站无严格反爬，避免一秒一页即可）。

#### 7. 【可选：调度 / 可视化】**Airflow（定时爬取） + ECharts（职位数据可视化）**

- 定时爬取：用 Airflow 实现每日自动爬取各大厂商招聘站，更新职位数据；
- 可视化：把爬取的职位数据做可视化（薪资分布 / 岗位类型 / 地域分布），提升项目价值（简历加分项）。

### 步骤 3：职位 / 简历信息**向量化**（Ollama Embedding）

核心：把**简历的核心匹配字段**（求职意向 + 技术栈 + 实习 / 项目经历 + 求职地点）和**职位的核心字段**（职位名 + 任职要求 + 技术栈 + 工作地点），通过 Ollama 自带的 Embedding 模型转为**等维向量**，为后续相似度检索做准备。

#### 关键逻辑

- 只对**核心匹配字段**做向量化，剔除无关信息（如职位发布时间、简历个人信息），提升检索精准度；
- 复用 Ollama 本地 Embedding（无需额外部署），一行调用即可生成向量。

### 步骤 4：向量库检索，获取**高相似度职位**（Chroma DB）

核心：提前将 MySQL 中的职位信息**预处理向量化**，存入 Chroma 向量库（建立「职位 ID - 职位信息 - 职位向量」的映射）；用户简历向量生成后，直接在 Chroma 中做**余弦相似度检索**，快速返回 Top-N 高相似度职位（如 Top10）。

#### 关键优势

- 比直接在 MySQL 中模糊查询**精准 10 倍 +**，能识别语义相似（如 “大模型应用开发” 和 “AI 工程化开发”）；
- 检索速度极快，几千条职位数据**毫秒级返回**。

#### 核心代码（Chroma，一行安装：`pip install chromadb`）

### 步骤 5：AI 深度匹配分析，生成**匹配度 + 分析理由**（本地 Ollama）

核心：纯向量检索只能得到**相似度分数**，缺少 “人性化分析”；将**简历结构化信息 + 高相似度职位信息**一起传给本地 Ollama，让 AI 做**深度语义匹配**，输出**具体匹配度（百分制）+ 匹配理由（核心契合点）+ 待提升点**，让结果更有参考价值。

#### 关键 Prompt 设计

- 强制**百分制匹配度**，理由分「核心契合点」和「待提升点」，结构清晰；
- 基于技术栈、求职意向、经历、学历、地点做**多维度匹配**，避免单一维度判断。

### 步骤 6：匹配结果展示（极简 Streamlit，无需前端知识）

核心：快速做一个**可视化前端页面**，支持用户上传 PDF 简历、展示 Top10 高匹配职位，每一个职位都带**百分制匹配度 + AI 分析理由 + 职位详情**，用户体验友好，**5 行代码就能实现基础展示**。
